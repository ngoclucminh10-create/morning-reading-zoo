# 早读神器 - 部署指南

## 如何分享给其他人使用

由于浏览器安全限制，麦克风访问需要 HTTPS 连接。以下是几种部署方法：

### 方法1：GitHub Pages（推荐，免费）

1. **创建 GitHub 账户**
   - 访问 https://github.com 注册

2. **创建新仓库**
   - 点击 "New repository"
   - 仓库名称：`morning-reading-zoo`
   - 选择 Public（公开）
   - 勾选 "Add a README file"

3. **上传文件**
   - 点击 "Add file" -> "Upload files"
   - 拖拽或选择以下文件：
     - `web-version.html`（重命名为 `index.html`）
     - `styles/` 文件夹
     - `scripts/` 文件夹
     - `images/` 文件夹（如果有）
   - 提交更改

4. **启用 GitHub Pages**
   - 进入仓库的 Settings
   - 找到 Pages 部分
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "main"，文件夹选择 "/(root)"
   - 点击 Save

5. **获取链接**
   - 等待几分钟
   - 在 Pages 部分会看到你的网站地址
   - 格式：`https://[你的用户名].github.io/morning-reading-zoo`

### 方法2：Netlify（推荐，免费）

1. 访问 https://netlify.com 注册

2. 拖拽项目文件夹到部署区域

3. 自动生成 HTTPS 链接

### 方法3：Vercel（推荐，免费）

1. 访问 https://vercel.com 注册

2. 导入 GitHub 仓库或直接上传

3. 自动部署，生成 HTTPS 链接

### 方法4：使用 ngrok（临时分享）

1. 下载 ngrok：https://ngrok.com/download

2. 启动本地服务器：
   ```cmd
   run-server.bat
   ```

3. 在新的命令行窗口中：
   ```cmd
   ngrok http 8080
   ```

4. 复制生成的 HTTPS 链接分享给他人

## 注意事项

1. **HTTPS 要求**：所有现代浏览器都要求 HTTPS 才能访问麦克风

2. **用户交互**：首次使用需要用户点击"允许"麦克风权限

3. **移动设备**：在手机上也能使用，但确保使用 HTTPS 链接

4. **浏览器兼容性**：
   - Chrome 60+
   - Firefox 55+
   - Safari 11+
   - Edge 79+

## 本地测试 HTTPS

如果想在本地测试 HTTPS，可以使用：

```cmd
# 使用 mkcert 创建本地证书
npm install -g mkcert
mkcert -install
mkcert localhost 127.0.0.1 ::1

# 然后使用支持 HTTPS 的服务器
python -m http.server 8080 --bind localhost --cert localhost.pem --key localhost-key.pem
```

## 简单分享步骤总结

最简单的方式：
1. 上传到 GitHub Pages
2. 分享生成的 HTTPS 链接
3. 其他人点击链接即可使用

## 企业内网部署

如果在公司内部分享：
1. 使用公司内网服务器
2. 配置 SSL 证书
3. 或使用内网穿透工具如 frp、ngrok
